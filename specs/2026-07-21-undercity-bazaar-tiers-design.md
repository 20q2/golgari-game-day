# Undercity — Tier-Gated Bazaars & Island Endgame Bazaar

**Date:** 2026-07-21
**Status:** Design (approved)

## Problem

Every bazaar currently sells Tier-3 gear as freely as commons. In
[`_gen_shop_stock`](../infrastructure/lambda/undercity_db.py) each gear slot is
filled with `rng.choice(by_slot[s])` — a uniform pick across **all** tiers
(T1/T2/T3). With all five biome bazaars stocking T3 at ~⅓ of every slot, players
buy top-tier gear early and cheaply, which trivializes the mid-game climb and the
combat balance that assumes T3 is an endgame payoff.

## Goal

- Make T3 gear an **endgame purchase**, sold at a single central-island bazaar.
- Keep the five biome bazaars to T1/T2, with T3 appearing only as a **rare
  black-market event**.
- Gear only. Grimoires stay Tier-1 at all bazaars (as today); consumables are
  untiered.

## Decisions (from brainstorming)

| Question | Decision |
|---|---|
| Island bazaar placement | **Filler node `isl_bg1`** — convert one of the two decorative "mystery" spaces beside the boss into a `shop`. |
| Island gear mix | **Mostly T2, some T3** — weighted per slot ~70% T2 / ~30% T3. |
| Biome-bazaar T3 | **Rare "black-market" roll** — biome bazaars are T1/T2, with a low-probability event that injects one T3 line. |
| Scope | **Gear only** — grimoires stay T1, consumables untiered. |
| Trading post | **Untouched.** Stays at `isl_trade` exactly as-is. The user is repurposing it separately (see Out of scope). |

## Backend changes

### Config (`infrastructure/lambda/undercity_config.py`)

```python
# Gear tiers a standard (biome) bazaar may stock — uniform pick among all
# pieces of these tiers within the chosen slot.
BAZAAR_GEAR_TIERS = {1, 2}

# Island bazaar gear tiers as {tier: weight} for the per-slot pick.
# ~70% T2 / ~30% T3 → "mostly T2, some T3".
ISLAND_BAZAAR_GEAR_TIERS = {2: 7, 3: 3}

# Per-(node, window) chance a biome bazaar rolls a rare "black-market" event
# that forces one of its gear slots to a T3 piece. 30-min windows → ~1 sighting
# per bazaar every ~10 hours. "Very rare event."
BAZAAR_BLACKMARKET_CHANCE = 0.05
```

### Data (`infrastructure/lambda/undercity_data.py`)

```python
# Bazaar nodes that stock the endgame (T2/T3) table instead of the biome table.
ISLAND_BAZAAR_NODES = {'isl_bg1'}
```

`_gen_shop_stock` receives only the node id (a string), so the island set is keyed
by id.

### `_gen_shop_stock` (`infrastructure/lambda/undercity_db.py`)

Re-index gear by slot **and** tier, then pick tier-first:

1. Build `by_slot_tier[slot][tier] = [gid, ...]` from `data.GEAR`.
2. `is_island = node in data.ISLAND_BAZAAR_NODES`.
3. Shuffle slots, take `SHOP_GEAR_SLOTS` of them (unchanged).
4. **Biome bazaars:** roll `rng.random() < BAZAAR_BLACKMARKET_CHANCE`; if it fires,
   pick one of the chosen slots as the black-market slot. For each slot: the
   black-market slot forces `tier = 3` (if that slot has a T3 piece), otherwise
   pick uniformly among all pieces whose tier ∈ `BAZAAR_GEAR_TIERS`.
5. **Island bazaar:** for each slot, pick a tier by weight from
   `ISLAND_BAZAAR_GEAR_TIERS` (restricted to tiers that slot actually has),
   then a random piece of that tier.
6. Each gear line carries `blackMarket: True` only on the rare biome T3 line;
   `False`/absent otherwise. The client badges it.

Ordering of `rng` calls is fixed and identical for all players, preserving the
"deterministic per `(node, window)`, no coordinated write" invariant. The
black-market roll is consumed before slot picks so it stays deterministic.

Consumables and grimoires blocks are unchanged. Grimoires remain
`GRIMOIRES[g]['tier'] == 1`.

### Map (`infrastructure/lambda/map.json` → sync)

- Change node `isl_bg1` from `"type": "mystery"` to `"type": "shop"`. Position and
  neighbors unchanged (it stays wired to `isl_ossuary` and `boss`).
- `isl_trade` (trading post) and `isl_bg2` (the other filler mystery) are left
  exactly as they are.
- Run `python infrastructure/lambda/sync_map.py` to refresh
  `public/data/undercity-map.json` (a pytest fails while the copies differ).

## Client changes

- **Map copy:** regenerated by `sync_map.py`.
- **Rendering:** `isl_bg1` now resolves as a `shop`. Confirm
  `board-terrain.ts` (`n.type === 'shop'`) and `board-tab.component.ts`
  (`ev.type === 'shop'`) render it as a bazaar (previously an inert `mystery`
  decal). The trading post at `isl_trade` is unaffected.
- **Keeper flavor** (`board-tab.component.ts`):
  - Biome bazaars rotate between **keeper 1** (`shopkeeper1.png`, turnips) and
    **keeper 2** (`shopkeeper2.png`, "the stock improved") only. The Witch drops
    out of the biome rotation.
  - The island bazaar (`isl_bg1`) always shows **the Witch, keeper 4**
    (`shopkeeper4.png`, "Baba has cauldrons…") — its fixed endgame vendor.
  - `bazaarKeeper()` becomes node-aware: return the Witch when the current bazaar
    is an island node, otherwise rotate over the biome pair by window index.
  - `tradingKeeper` (`shopkeeper3.png`, the ooze) is untouched — that persona is
    Umori, reserved for the future wandering trading post.
- **Black-market badge:** in the shop modal, mark any gear line with
  `blackMarket` (badge/label) so the rare biome T3 reads as an event.
- No tier data to mirror — `src/app/undercity/data/items.ts` already carries
  `tier` on every gear piece.

## Out of scope (future work)

- **Umori, the wandering trading post.** Separately, the user plans to turn Umori
  (the ooze shopkeeper) into a wandering NPC that lives in the wilderness, is
  visible in the overworld, shows a relocation timer, moves to a random location
  on that timer, and — when landed on — opens the trading-post feature seeded to
  **always start with T3 gear** (justified by how hard it is to reach). That is a
  distinct feature (overworld NPC + server-side wander tick + relocation timer +
  landing detection + trading-post T3 seeding) and gets its own spec → plan cycle.
  This design does **not** touch the trading post, `TRADING_POST_SEED`, or Umori.

## Non-goals

- No change to loot/trove drop tiers, evolution, or renown shop.
- No T2/T3 grimoire acquisition (still deferred).
- No new UI beyond the black-market badge and island keeper flavor.

## Balance rationale

- Biome bazaars: T1/T2 keep the early/mid climb on-curve; T3 is a ~1-in-20-window
  surprise, so a lucky sighting feels like a find rather than a shortcut.
- Island bazaar: reachable via the warp into the endgame hub, so a reliable T3
  supply only exists once players are already in the late game — exactly where T3
  power is intended.
