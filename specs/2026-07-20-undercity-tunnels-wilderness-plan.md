# Undercity Tunnels & Wilderness Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Split Undercity surface travel into two tier-dependent layers — Tier-1-only `tunnel` shortcuts between adjacent biomes, and a new open-to-all `wilderness` central-hub region that reconnects all biomes for evolved (Tier-2+) units who are locked out of the tunnels.

**Architecture:** The pure movement engine (`undercity_engine.py`) gains a `blocked` node-set parameter that callers populate from the mover's `tier`. The 10 existing `t_*` spur nodes are retyped to `tunnel` (safe passage, no landing event). A new `wilderness` region (~14 hand-authored nodes) is added to `map.json` as a hub-and-spoke crossroads so blocking tunnels never strands evolved units. The Angular board renderer learns the new types and greys tunnels out for evolved players.

**Tech Stack:** Python 3.11 Lambda (pytest, in-memory FakeTable suite), Angular 20 standalone components, canvas board renderer (TypeScript). Map source of truth is `infrastructure/lambda/map.json`, mirrored to `public/data/undercity-map.json` via `sync_map.py`.

**Spec:** `specs/2026-07-20-undercity-tunnels-wilderness-design.md`

**Run all backend tests with:** `cd infrastructure/lambda && python -m pytest tests -q`

```python
def legal_destinations(nodes: dict, start: str, steps: int,
                       closed: frozenset = frozenset(),
                       blocked: frozenset = frozenset()) -> set:
    """
    Dokapon exact-count movement: every node reachable in exactly `steps`
    edges without immediately reversing the previous edge. Dead-end branches
    shorter than the roll simply contribute nothing. `closed` holds sealed
    barrier nodes: a walk that reaches one STOPS at it (spending the rest of
    the roll). `blocked` holds nodes you may never step ONTO (e.g. tunnels for
    evolved units) — they are never a destination and never a corridor. The
    start node is never treated as blocked, so you can always walk off one.
    """
    results = set()
    stack = [(start, None, steps)]
    seen = set()
    while stack:
        node, prev, remaining = stack.pop()
        if remaining == 0:
            results.add(node)
            continue
        if node in closed and node != start:
            results.add(node)
            continue
        key = (node, prev, remaining)
        if key in seen:
            continue
        seen.add(key)
        for nb in nodes[node]['neighbors']:
            if nb == prev:
                continue
            if nb in blocked:
                continue
            stack.append((nb, node, remaining - 1))
    return results
```

(Only the signature/docstring and the added `if nb in blocked: continue` line change; keep the trailing comment block as-is.)

```python
def board_distance(nodes: dict, start: str, goal: str, max_steps: int,
                   closed: frozenset = frozenset(),
                   blocked: frozenset = frozenset()) -> int | None:
    """
    Shortest hop count from start to goal, or None past max_steps. Plain BFS.
    `closed` (sealed barriers) blocks passage but may be the goal itself.
    `blocked` holds nodes you may never step onto (never a corridor, never a
    goal you can arrive at).
    """
    if start == goal:
        return 0
    frontier = {start}
    seen = {start}
    for dist in range(1, max_steps + 1):
        nxt = set()
        for node in frontier:
            if node != start and node in closed:
                continue
            for nb in nodes[node]['neighbors']:
                if nb in seen or nb in blocked:
                    continue
                if nb == goal:
                    return dist
                seen.add(nb)
                nxt.add(nb)
        frontier = nxt
        if not frontier:
            break
    return None
```
